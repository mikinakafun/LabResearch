
## 概要

このVaultは卒業研究の知識・思考・成果を一元管理するためのもの。  

目的は以下の3点に限定する：

  

- 知識の蓄積（論文・資料）

- 思考の発展（メモ・アイデア）

- 成果の生成（論文・発表）

  

---

  

## ディレクトリ構造
  

├── README.md

├── keywords.md

├── inbox/

├── notes/

├── literature/

├── ideas/

├── outputs/

└── assets/
- `inbox/`：未整理メモ（とにかくここに入れる）
- `notes/`：自分の理解・要約（最重要）
- `literature/`：論文・参考資料の整理
- `ideas/`：仮説・研究テーマ・発想
- `outputs/`：論文・スライド・発表資料
- `assets/`：画像・PDFなどの添付ファイル
- `keywords.md`：用語・タグの定義

---

## 基本運用フロー

### 1. 入力（Capture）
- 思いつき・読書メモ・授業内容 → `inbox/`

### 2. 整理（Process）
- inboxを定期的に処理
- 以下に振り分け：
  - 理解 → `notes/`
  - 論文 → `literature/`
  - 仮説 → `ideas/`

### 3. 接続（Link）
- ノート同士をリンク
- keywordsを使ってタグ付け

### 4. 出力（Output）
- `notes/`と`ideas/`から`outputs/`を生成

## keywordsの使い方

- 重要概念のみ登録
- 同義語をまとめる
- タグとして再利用する

例：
	# **memory**

	- episodic memory
	    
	- odor-evoked memory

---

## 設計思想

- フォルダは最小限（分類ではなく役割）
- すべてはリンクで繋ぐ
- 完璧に整理しない（検索前提）

---

## 注意

- inboxを放置すると崩壊する
- ノートを増やすだけでは意味がない（必ずリンクする）
- outputsを作らないと研究にならない

---

## 行動指針

1. 思考はすべてinboxに投げる  
2. 定期的に整理する  
3. 必ずリンクする  
4. outputsを継続的に生成する
